# 集合竞价三合一策略 - PTrade 版本（纯文本规格）

## 一、策略概述
- 交易品种：A股（不含ST、科创、北交所，上市>250天）
- 周期频率：日线（定时任务）
- 核心逻辑：三个子策略合并——一进二集合竞价 + 首板低开 + 弱转强

## 二、子策略信号规则

### 子策略1：一进二集合竞价
1. 昨日涨停（非连续涨停，前日未涨停）
2. 近N日最高价位置 ≤ 50%
3. 昨日成交额 5.5亿~20亿
4. 今日竞价涨幅 / (涨停价/1.1) ∈ (1.0, 1.06]
5. 上市 > 250天，市值 70亿~520亿
6. 近N日量能突破（今日成交量 > 前N日最大成交量 × 0.9）

### 子策略2：首板低开
1. 昨日涨停（非连续涨停）
2. 近10日无连续涨停
3. 近60日最低价位置 ≤ 50%
4. 今日开盘价 / 昨日收盘价 ∈ [0.955, 0.97]
5. 昨日成交额 ≥ 1亿

### 子策略3：弱转强
1. 4日内涨幅 ≤ 28%
2. 昨日涨幅 > -5%（非大阴线）
3. 昨日成交额 3亿~19亿
4. 昨日均价涨幅 > -4%
5. 上市 > 250天，市值 70亿~520亿
6. 近N日量能突破
7. 竞价涨幅 / (涨停价/1.1) ∈ [0.98, 1.09)

## 三、买入与卖出规则

### 买入（9:26）
- 三个子策略并集去重
- 等权分配资金，每只股票分配现金/标的数量
- 只在可用资金比例 > 30% 时买入

### 卖出（11:25 / 14:50）
- 11:25：成本价 < 当前价 < 涨停价 → 止盈卖出
- 14:50：
  - 跌破5日线 → 止损卖出
  - 或 成本价 < 当前价 < 涨停价 → 止盈卖出

## 四、参数配置

## 五、时间轴
| 时间 | 动作 |
|------|------|
| 09:01 | 选股 |
| 09:26 | 买入 |
| 11:25 | 止盈卖出 |
| 14:50 | 止损/止盈卖出 |

## 六、参考对比（对照 etf23）
- 与 etf23 相同点：T+1检查、防重复下单机制、涨跌停检查、实盘回调、pending_order防重复
- 与 etf23 差异点：etf23是ETF轮动；本策略是三个股票选股策略合并；本策略有ST过滤+新股过滤+市值筛选

## 七、PTrade API 差异点（JoinQuant → PTrade）
| JoinQuant | PTrade |
|-----------|--------|
| context.current_dt | context.blotter.current_dt |
| context.previous_date | _get_previous_trading_day(context) |
| pos.closeable_amount | pos.enable_amount |
| order_value(code, cash) | order(code, shares, limit_price=price) |
| set_option/log.set_level | 删除 |

## 八、文件结构
- 状态持久化：无（不复用昨日数据，每日独立选股）
- 配置读取：无外部配置
