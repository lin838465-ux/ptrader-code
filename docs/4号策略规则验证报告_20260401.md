# 4号策略规则验证报告

日期：2026-04-01

## 数据基础

本报告只使用已经落到本地的证据库进行验证，不依赖临时联网推断。

本地数据位置：

- `data/4hao_focused/trades/trades_clean.csv`
- `data/4hao_focused/daily_windows/*.csv`
- `data/4hao_focused/minute/*.csv`
- `data/4hao_focused/derived_features.csv`

本次验证覆盖：

- 交割单交易：72 笔
- 有效平仓交易：68 笔
- 买入日分钟数据：72 份
- 卖出日分钟数据：68 份
- 分钟数据粒度：5 分钟

注意：

- `T048 / 601012 隆基绿能` 的交割单存在明显异常，记录为先卖后买，不能作为卖出逻辑证据。
- 仍未卖出的 3 笔交易不参与卖出规则验证。
- 日线窗口是“买入前 10 个自然日到卖出日”，因此少量样本的“买入前 5 个交易日”会因为窗口不够长而无法完整判断。

## 一、买入规则验证

### 1. 股票池范围：高置信

72 笔交易全部是主板代码：

- `600`
- `601`
- `603`
- `605`
- `000`
- `002`
- `003`

没有看到：

- 创业板 `300`
- 科创板 `688`

结论：

**股票池高度疑似限定在主板，不是泛全市场，也不是传统“小市值最低流通市值轮动”。**

### 2. 买入时间：高置信

交割单中的 72 笔买入时间全部为 `09:30`。

结合你后来给的实盘日志：

- `09:10` 盘前出候选池
- `09:27` 执行最终买入

结论：

**真实策略应是集合竞价到开盘瞬间完成筛选与下单，交割单统一记在 09:30。**

### 3. “涨停记忆 + 前一日回调”：高置信

在本地数据中，真正能完整看到买入前 5 个交易日的样本共有 62 笔。

其中：

- `62 / 62` 在买入前 5 个交易日内出现过至少一次涨停
- `61 / 62` 同时满足“前一日回调”

唯一明确例外：

- `T020 / 603712 / 七一二`
- 2025-11-28 涨停
- 2025-12-01 直接高开买入
- 前一日并不是回调日

结论：

**“前 5 个交易日内至少有一次涨停”是硬规则。**

**“买入前一日出现回调”是主规则，但不是 100% 无例外。**

### 4. 低开低吸：高置信

在 68 笔有效平仓交易中：

- `67 / 68` 买入日开盘价低于前一日收盘价
- `56 / 68` 的开盘缺口位于 `-3% ~ 0%`
- `64 / 68` 位于 `-6% ~ 0%`
- 只有 `1 / 68` 是正缺口买入
- 严重低开 `<= -6%` 的仅 `3 / 68`

开盘缺口统计：

- 均值：`-1.83%`
- 中位数：`-1.20%`
- 25% 分位：`-2.28%`
- 75% 分位：`-0.51%`

唯一明显的正缺口例外仍然是：

- `T020 / 七一二`

结论：

**“低开低吸”不是口号，而是数据上非常硬的特征。**

更准确地说，策略偏好的是：

**轻微低开，而不是大幅闷杀低开。**

### 5. 开盘后第一根 5 分钟：中置信

对 72 笔买入日第一根 5 分钟 K 线统计：

- `47.2%` 第一根 5 分钟收涨
- `86.1%` 第一根 5 分钟最高价高于开盘价
- `29.2%` 第一根 5 分钟收盘相对开盘上涨至少 `1%`

结论：

**买入后并不要求第一根 5 分钟立刻强势收涨。**

但大多数样本在第一根 5 分钟里都会出现向上承接，说明它更像“低开后抢回流”的票，而不是纯弱势低开股。

## 二、卖出规则验证

### 1. 盈利单围绕 5% 止盈：高置信

有效平仓交易中：

- 盈利单：47 笔
- 亏损单：21 笔

盈利单收益率分布：

- 最小值：`+4.96%`
- 中位数：`+5.53%`
- 均值：`+6.19%`

进一步看持有期间的日线最高价：

- `46 / 47` 盈利单，在持有期内至少触达过 `+5%`
- 剩余 1 笔是 `T012 / 海思科`
- 它的持有期最高涨幅是 `+4.995%`

结论：

**止盈线非常接近 `+5%`，这基本可以视为硬规则。**

### 2. 亏损单几乎都在尾盘处理：高置信

21 笔亏损单中：

- `21 / 21` 都在 `14:55 ~ 14:59` 之间卖出

亏损单收益率分布：

- 最大值：`-4.84%`
- 中位数：`-6.52%`
- 均值：`-6.68%`

这说明两件事：

1. 止损不是盘中即时触发
2. 即使止损线名义上写的是 `-5%`，实际成交结果会因为拖到尾盘而出现 `-6%`、`-7%` 甚至更差

结论：

**“止损只在 14:55 左右处理”是非常强的硬规则。**

### 3. 亏损单大多在卖出日已经跌破 -5%：高置信

用卖出日 5 分钟数据验证，21 笔亏损单中：

- `19 / 21` 在卖出时点之前，卖出日最低价已经跌破买入价的 `-5%`

剩余 2 笔例外：

- `T002 / 联创光电`
- `T023 / 和而泰`

这两笔外部行情源与交割单成交价存在明显差异，说明数据源存在个别价格偏差，不能据此否定尾盘止损逻辑。

结论：

**从行为上看，策略确实是在尾盘对“已落入止损区间”的持仓进行集中处理。**

### 4. 盈利单卖出时段：高置信

47 笔盈利单中：

- `68.1%` 在 `09:30 ~ 09:59` 卖出
- 仅 2 笔在 `14:55 ~ 14:57` 卖出

这两笔尾盘盈利单分别是：

- `T012 / 海思科 / +5.00%`
- `T013 / 中新集团 / +5.01%`

结论：

**盈利单通常在早盘或盘中一旦达到目标就兑现，不会像亏损单那样拖到尾盘。**

## 三、持仓周期验证

68 笔有效平仓交易的持仓交易日统计：

- 中位数：2 天
- 均值：2.96 天
- 75% 分位：3 天

结论：

**这与“平均 2-3 天”的截图描述是吻合的。**

## 四、例外与脏数据

### 1. 明确的结构性例外

最值得关注的真实例外是：

- `T020 / 七一二`

它满足“涨停记忆”，但不满足“前一日回调”，而且还是正缺口买入。

这说明：

**策略主框架非常稳定，但很可能允许少量强势特例。**

### 2. 明确的异常交割单

- `T048 / 601012 隆基绿能`

交割单写成了先卖后买，不应参与规则验证。

### 3. 局部数据源偏差

个别样本中，交割单成交价与外部行情源的当日极值存在偏差，例如：

- `T002 / 联创光电`
- `T023 / 和而泰`

这提醒我们：

**阈值应该按“接近 5% / -5% 的行为规则”理解，而不是机械卡死到小数点后两位。**

## 五、当前最可靠的可编码版本

基于本地数据，当前最可靠的编码框架是：

### 买入

1. 主板股票池
2. 过滤 ST、停牌、退市
3. 最近 5 个交易日内至少出现过一次涨停
4. 优先要求前一交易日为回调日
5. 当天竞价或开盘轻微低开
6. 严重低开不买
7. 每天只买 1 只

### 卖出

1. 盘中达到约 `+5%` 即止盈
2. 若未止盈，亏损单只在 `14:55` 左右统一检查
3. 尾盘若达到止损条件则卖出
4. 持仓周期通常为 2 到 3 天，可能存在超时卖出

## 六、结论分级

### 高置信规则

- 主板池
- 开盘买入
- 最近 5 日有涨停记忆
- 轻微低开低吸
- 盈利围绕 `+5%` 止盈
- 亏损在 `14:55` 左右处理

### 中置信规则

- 前一日回调几乎总是成立，但存在少数特例
- 可能存在“强势例外票”放宽条件
- 可能存在持仓天数上限

### 低置信、仍需继续验证

- 候选股在盘前是按什么排序
- 最终为什么在同一天的多个候选中只选某一只
- 是否叠加了成交量、竞价金额、换手率、封板质量等条件

## 七、下一步建议

建议后续工作按这个顺序推进：

1. 先按本报告的高置信规则写出一版“4号策略验证版”
